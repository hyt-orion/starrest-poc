# 星憩时刻 · POC 后下一步路线图

> 日期：2026-07-16
> POC 状态：P0+P1 全部完成，永久部署在 https://hyt-orion.github.io/starrest-poc/

---

## 核心判断

**POC 验证了"技术上能做"，下一步要验证"家长真的会用、真的能减负"。**

文档调研报告的结论：
- ✅ 需求真实（ZBI 42.8/88，严重耗竭）
- ✅ 技术可行（除行为分类器外全可复用开源轮子）
- ⚠️ **AI 减负实证空白**——"有方向无确证"，需要自行补效果测量
- ⚠️ 隐私是最大落地变量（已做首屏+本地处理，需验证家长接受度）

---

## 第一阶段：种子用户试用（最高优先级，1-2 周）

### 做什么
找 **3-5 个星宝家长**，让他们用 POC 1-2 周，观察和收集反馈。

### 怎么找
- 绿房子公益的家长群（文档种子用户 = 绿房子机构家长）
- 康复机构推荐
- 家长社群（微信群/公众号）

### 收集什么反馈
| 维度 | 关键问题 |
|---|---|
| 模型准确性 | 活跃指数准不准？有没有漏报（孩子哭了但没报警）？有没有误报（正常活动但报警了）？ |
| 悬浮球体验 | 平时安安静静吗？干预级脉冲够不够明显？会不会打扰？ |
| 喘息活动 | 冥想呼吸法有用吗？树洞会倾诉吗？用几次？每次多久？ |
| 隐私信任 | 看完隐私首屏后放心吗？敢不敢开着摄像头去做别的事？ |
| 整体感受 | 真的能"喘口气"吗？会比不用的时候更放松吗？ |

### 关键假设要验证
> "AI 替家长看着孩子，让家长真的拥有属于自己的时间" —— 这个假设成立吗？

## 第二阶段：效果测量（与第一阶段并行）

文档明确建议的自证减负 RCT：

| 指标 | 工具 | 时间点 |
|---|---|---|
| 照护负担 | ZBI（Zarit 照护负担量表） | 试用前 + 试用 2 周后 |
| 睡眠质量 | PSQI（匹兹堡睡眠质量指数） | 同上 |
| 育儿压力 | PSI（育儿压力指数） | 同上 |
| 使用数据 | App 内置日志（次数/时长/干预触发次数） | 自动记录 |

> 这组数据是将来找资金、找合作、找机构背书的**硬证据**。没有它，"AI 减负"只是说法，不是事实。

## 第三阶段：技术迭代（根据种子用户反馈排优先级）

### P2 — 上线准备

| 项 | 说明 | 何时做 |
|---|---|---|
| 后端 | Cloudflare Workers + D1 + R2（基线聚合/树洞API/激励API） | 种子用户反馈需要多设备同步时 |
| 激励体系 | 盲盒(canvas-confetti) + 累计里程碑 | 种子用户反馈"用了几次就不想用了"时 |
| 无障碍 | VoiceOver/TalkBack 全覆盖 | 有视障家长用户时 |
| 行为分类器 | 自闭症刻板行为检测（RTMPose+GB） | 通用姿态不够准时 |
| WASM 本地化 | 如果 jsdelivr CDN 不稳定 | 用户反馈 WASM 加载失败时 |
| Worker 推理 | 解决 WASM 在 Worker 里的加载问题 | 主线程卡顿影响体验时 |

### 延后
- 声音克隆（需伦理框架 + 深度合成规管）
- AI 引导星宝结构化活动（参考 Stanford Noora，人在回路）

## 建议的执行顺序

```
现在 ──→ 找 3-5 个家长试用（1-2 周）
         │
         ├── 并行：ZBI/PSQI 前测
         ├── 并行：收集使用反馈
         │
2 周后 ──→ 根据反馈排 P2 优先级
         │
         ├── 后端？激励？无障碍？行为分类器？
         │
4 周后 ──→ ZBI/PSQI 后测 + 效果报告
         │
         ├── 有数据 → 找资金/合作/机构背书
         │
6-8 周 ──→ 备案域名 + 真实登录 + 合规
         │
         → 正式发布
```

> **最重要的一件事**：别在技术迭代上花太多时间，先让真实家长用起来。POC 已经够用了——技术不是瓶颈，"家长真的会用吗"才是。

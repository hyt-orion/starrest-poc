## 一、看护核心

### 1.1 MoveNet 推理性能优化
- **问题**：TFJS 库 2.7MB，首次加载慢；MoveNet 推理在主线程，5fps 时可能卡顿
- **方案**：用 `@tensorflow/tfjs-core` + `@tensorflow/tfjs-backend-webgl` 替代完整 TFJS 包，减少 bundle 体积；或用 dynamic import 延迟加载 TFJS
- **预期效果**：JS 从 2.7MB 降到 ~1MB，首次加载快 2-3 倍

### 1.2 自适应帧率
- **问题**：当前固定 5fps（200ms），不管孩子是否在动
- **方案**：活跃指数低时降到 2fps 省电，活跃指数高时升到 8fps 提高灵敏度
- **预期效果**：功耗降低 40%，异常时响应更快

### 1.3 行为分类器升级
- **问题**：当前用规则（过零率/距离/交替频率），准确率有限
- **方案 A**：加更多特征（关键点速度、对称度、周期性 FFT）
- **方案 B**：用 TensorFlow.js 训练轻量分类模型（MobileNet 迁移学习）
- **预期效果**：行为识别准确率从 ~70% 提升到 ~90%

### 1.4 音频分类升级（YAMNet 替代 RMS）
- **问题**：当前只用 RMS 能量，不能区分哭声/尖叫/正常说话
- **方案**：用 @tensorflow-models/speech-commands 或 YAMNet TFJS 版做音频分类
- **预期效果**：能区分哭声/尖叫/撞击/说话，活跃指数更精准

### 1.5 多人检测
- **问题**：当前只检测 1 个人，如果画面里有其他人（家长/兄弟姐妹）会干扰
- **方案**：MoveNet 改为 MULTIPOSE 模式，跟踪所有人，只对"主要人物"（最大面积）做行为分析
- **预期效果**：减少误报

## 二、两端分离与通信

### 2.1 WebSocket 实时通信（Durable Object）
- **问题**：localStorage 只能同浏览器跨标签，不能跨设备
- **方案**：用 Cloudflare Durable Object + WebSocket，星宝端推流，家长端订阅
- **预期效果**：真正跨设备，家长手机看星宝平板的画面

### 2.2 WebRTC P2P 视频传输
- **问题**：当前用 JPEG dataURL 截帧传输，画质低、延迟大
- **方案**：用 WebRTC P2P 直接传输视频流（不需要中转服务器存储视频）
- **预期效果**：30fps 实时视频，画质大幅提升，延迟 <500ms

### 2.3 连接状态管理
- **问题**：星宝端断线后家长端不知道
- **方案**：心跳机制（每 5 秒发 ping），超时 15 秒判定断线，家长端显示"星宝端已断开"
- **预期效果**：断线感知 + 自动重连提示

### 2.4 房间码配对
- **问题**：当前任何同一浏览器的标签页都能配对，没有身份验证
- **方案**：星宝端生成 4 位房间码，家长端输入房间码配对，只有配对的设备才能通信
- **预期效果**：安全性提升，防止误连

## 三、喘息活动

### 3.1 冥想引导升级
- **问题**：当前只有 4-7-8 呼吸法动画，没有音频引导
- **方案 A**：用 Web Speech API（浏览器原生 TTS）生成中文语音引导
- **方案 B**：预录冥想音频，存 R2，按场景播放（3分钟/5分钟/10分钟）
- **预期效果**：家长闭眼跟随语音，不用看屏幕

### 3.2 树洞升级
- **问题**：当前只有文字，没有情感筛选
- **方案**：加 AI 情感分析（TFJS 情感分类），负面情绪帖子加温暖提示；加"拥抱"按钮（别人可以给倾诉一个拥抱）
- **预期效果**：更温暖的社区氛围

### 3.3 手作功能实现
- **问题**：当前只是"即将上线"占位
- **方案**：接入手作教程视频（R2 存储），加材料清单 + 步骤计时器
- **预期效果**：第三个喘息活动可用

### 3.4 喘息活动计时与进度
- **问题**：当前没有计时，用户不知道做了多久
- **方案**：冥想/手作加倒计时（15/30 分钟），树洞加"已停留 X 分钟"提示
- **预期效果**：用户有掌控感

## 四、激励体系

### 4.1 连续打卡奖励梯度
- **问题**：当前每 5 次送心理咨询，梯度太粗
- **方案**：3 次→鼓励语音，7 次→治愈壁纸，14 次→心理咨询，30 次→专属称号
- **预期效果**：更多正反馈触点

### 4.2 喘息时长统计
- **问题**：当前只统计次数，不统计时长
- **方案**：记录每次喘息活动时长，每周汇总"本周喘息 X 分钟"
- **预期效果**：家长看到累计休息时间，有成就感

### 4.3 壁纸实际可用
- **问题**：当前"治愈壁纸"只是文字奖励，不能实际使用
- **方案**：解锁壁纸后可以下载/设为 App 背景
- **预期效果**：奖励有实际价值

### 4.4 鼓励语音实际播放
- **问题**：当前"鼓励语音"只是文字，不能播放
- **方案**：用 Web Speech API 朗读鼓励语，或预录音频
- **预期效果**：盲盒开箱后有声音反馈

## 五、隐私与安全

### 5.1 视频帧加密传输
- **问题**：当前 JPEG dataURL 明文存储在 localStorage
- **方案**：用 Web Crypto API 加密 dataURL，只有配对设备能解密
- **预期效果**：即使 localStorage 被读取也无法看到画面

### 5.2 自动模糊非看护区域
- **问题**：视频帧包含整个画面，可能有隐私敏感区域
- **方案**：用 MoveNet 关键点定位人体区域，只传输人体区域裁剪后的帧
- **预期效果**：减少隐私泄露面积，传输数据量更小

### 5.3 数据自动过期
- **问题**：IndexedDB 基线数据永久存储
- **方案**：基线数据 30 天自动删除（只保留最近 7 天），树洞帖子 7 天自动过期
- **预期效果**：减少数据留存风险

## 六、UX/UI 精进

### 6.1 悬浮球增强
- **问题**：当前悬浮球只显示数字和颜色
- **方案**：加最近 60 秒迷你折线图（活跃指数趋势）；异常行为时显示行为图标
- **预期效果**：家长一眼看到趋势和异常

### 6.2 深色/浅色主题切换
- **问题**：当前只有深色主题
- **方案**：加浅色主题选项（白天使用更舒适）
- **预期效果**：适配不同场景

### 6.3 家长端右半画面全屏
- **问题**：当前右半画面只有半屏，画面小
- **方案**：加"全屏"按钮，点击后右半画面全屏显示
- **预期效果**：看孩子更清楚

### 6.4 引导页跳过记忆
- **问题**：每次清缓存后引导页重新出现
- **方案**：引导完成状态也存后端（登录后从后端读取是否已看过）
- **预期效果**：换设备后不重复看引导

## 七、PWA 与离线

### 7.1 离线模式
- **问题**：当前 service worker 只缓存静态资源，离线时看护功能不可用
- **方案**：离线时仍可使用本地看护（单设备模式），在线时自动同步数据到后端
- **预期效果**：断网也能看护

### 7.2 桌面通知
- **问题**：当前干预级提醒只有悬浮球脉冲
- **方案**：用 Notifications API 发送系统级通知（即使浏览器在后台也能收到）
- **预期效果**：家长不会错过干预级提醒

### 7.3 后台保活
- **问题**：手机浏览器后台标签页可能被系统杀掉
- **方案**：用 Wake Lock API 保持屏幕常亮；PWA 安装后用 Background Sync
- **预期效果**：星宝端长时间运行不被杀

### 8.1 前端接入后端 API
- **方案**：apiClient.ts 已写好，需要改 authStore/settingsStore/rewardsStore/RelaxPanels 调后端

- **方案**：冥想音频存 R2，前端按需加载
- **预期效果**：喘息活动有真实音频引导

## 九、数据与分析

### 9.1 看护报告
- **问题**：家长看不到历史趋势
- **方案**：每天/每周生成看护报告（活跃指数趋势图、行为分类统计、喘息时长汇总）
- **预期效果**：家长了解孩子的状态变化

### 9.2 异常事件记录
- **问题**：干预级事件没有持久化记录
- **方案**：干预级触发时自动记录时间戳+活跃指数+行为分类，家长可回看
- **预期效果**：复诊时给医生看

### 9.3 效果测量量表
- **问题**：文档规划 ZBI/PSQI 前后测，但没有实现
- **方案**：在 App 内嵌入 ZBI 量表（12 题简化版），首次使用和每 2 周填写，自动对比
- **预期效果**：有数据证明"AI 减负"效果
